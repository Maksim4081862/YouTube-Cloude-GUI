# 🎬 YouTube-Cloud-GUI

**Бесконечное облачное хранилище на базе YouTube.**  
Преобразуйте любые файлы (`.exe`, `.zip`, `.docx` и др.) в видеофайлы, загружайте их на YouTube и восстанавливайте обратно без потерь. Поддерживает шифрование данных и имеет удобный графический интерфейс.

> ⚠️ **Внимание:** Этот проект является экспериментальным инструментом для обхода ограничений хранения данных. Используйте его на свой страх и риск. Нарушение условий использования YouTube может привести к блокировке аккаунта. Не храните критически важные данные единственным экземпляром.

---

## 🚀 Возможности

* **Любые форматы:** Кодирование исполняемых файлов, архивов, документов и т.д.
* **Без потерь:** Восстановление файла бит-в-бит благодаря специальному алгоритму кодирования цветов.
* **Шифрование:** Поддержка XOR-шифрования с пользовательским ключом (`key.txt`).
* **Два режима работы:**
  * 🖥️ **GUI:** Удобный графический интерфейс для новичков.
  * 💻 **CLI:** Работа через командную строку для скриптов и автоматизации.
* **Защита данных:** Встроенные маркеры начала/конца файла и защитные кадры.

## 📋 Требования

Для работы проекта необходимо установить Python 3.6+ и следующие библиотеки:

```bash
pip install opencv-python numpy
```

> **Примечание:** Остальные модули (`os`, `math`, `subprocess`, `tempfile`, `shutil`, `sys`, `re`, `hashlib`) входят в стандартную библиотеку Python и устанавливаются автоматически.

Для наилучшего качества сжатия видео рекомендуется также установить **FFmpeg** и добавить его в переменную среды PATH. Если FFmpeg не найден, программа использует встроенные средства OpenCV (кодировщик MP4V).

## 🛠️ Установка и запуск

### 1. Через графический интерфейс (Рекомендуется)

Запустите файл интерфейса:

```bash
python youtube_cloud_gui.py
```

В открывшемся окне вы сможете:

1. Выбрать файл для кодирования.
2. Указать путь сохранения видео.
3. Настроить шифрование (создать ключ).
4. Отслеживать прогресс и просматривать логи операций.
5. Декодировать скачанные видео обратно в файлы.

### 2. Через командную строку (CLI)

#### Кодирование файла

Чтобы превратить файл в видео:

```bash
python coder.py encode <ИМЯ_ФАЙЛА>.xxx <ИМЯ_ВИДЕО>.mp4
```

* `<ИМЯ_ФАЙЛА>.xxx` — путь к исходному файлу (например, `program.exe` или `archive.zip`).
* `<ИМЯ_ВИДЕО>.mp4` — имя выходного видеофайла.

**Пример:**

```bash
python coder.py encode my_secret_data.zip output_video.mp4
```

#### Декодирование видео

Чтобы восстановить файл из видео:

```bash
python coder.py decode <ИМЯ_ВИДЕО>.mp4 [ПАПКА_НАЗНАЧЕНИЯ]
```

* `<ИМЯ_ВИДЕО>.mp4` — путь к скачанному видеофайлу.
* `[ПАПКА_НАЗНАЧЕНИЯ]` — (необязательно) папка, куда сохранить восстановленный файл. По умолчанию используется текущая папка.

**Пример:**

```bash
python coder.py decode output_video.mp4 ./restored_files
```

## 🔐 Шифрование данных

Для защиты ваших данных от посторонних глаз реализована поддержка ключей шифрования.

1. Создайте в папке со скриптом файл с именем `key.txt`.
2. Запишите внутрь него любой текстовый ключ (пароль).
   
   ```text
   MySuperSecretPassword123
   ```
3. Сохраните файл.

**Как это работает:**

* При **кодировании**: если файл `key.txt` существует, данные будут зашифрованы перед превращением в видео.
* При **декодировании**: программа автоматически прочитает `key.txt` и попытается расшифровать данные.
  * ✅ Если ключ верный — файл восстановится корректно.
  * ❌ Если ключ неверный или отсутствует — вы получите поврежденный файл.

> 💡 Вы можете использовать один ключ для всех файлов или создавать разные `key.txt` для разных наборов данных (не забудьте переименовать файл ключа перед операцией).

## ⚙️ Технические детали

* **Разрешение видео:** 1920x1080 (Full HD).
* **Частота кадров (FPS):** 6 (оптимизировано для баланса между размером видео и скоростью обработки).
* **Алгоритм:** Данные разбиваются на биты, группируются по 4 бита и кодируются одним из 16 специальных цветов.
* **Надежность:** Используются маркеры концовки данных (`EOF`) и дублирование блоков внутри кадра для устойчивости к сжатию YouTube.

## ❓ FAQ

**Q: Можно ли хранить так вирусы?**  
A: Технически да, но загрузка вредоносного ПО на YouTube нарушает правила платформы и может привести к вечной блокировке канала. Используйте только для легальных данных.

**Q: Почему видео такое длинное?**  
A: Пропускная способность канала ограничена (~1.5 КБ данных на кадр). Для файла размером 10 МБ потребуется видео длительностью несколько минут.

**Q: Что будет, если YouTube изменит кодек?**  
A: Это основной риск метода. Изменение алгоритмов сжатия может сделать старые видео нечитаемыми. Всегда храните резервные копии важных файлов на локальном диске.
